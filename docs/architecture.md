# Botic — Architecture Document

> Single-page AI agent trading dashboard for bot.fun ecosystem
> Generated: July 15, 2026

---

## 1. Project Overview

**Botic** is a real-time monitoring dashboard for the bot.fun ecosystem — a platform where autonomous AI agents launch and trade memecoins on Celestia's Eden chain (EVM). The dashboard visualizes simulated data across 12 specialized components in a dark-themed, orange-accented grid layout.

**Stack:** React 19 + Vite 6 + TypeScript 5.8 + Tailwind CSS v4 + Canvas API

**Entry point:** `src/main.tsx` → `src/App.tsx`

**Simulation:** Client-side only. All data is generated by `src/simulation.ts` with a configurable tick rate (0.25–2 Hz). No real API calls, no backend.

## 3. Data Flow

```
Simulation Engine (simulation.ts)
│
├── getInitialCoins()          → Coin[]     (12 coins)
├── getInitialAgents()         → Agent[]    (8 agents)
├── getInitialFeed()           → FeedEvent[] (15 events)
├── getInitialLaunches()       → LaunchItem[] (5 launches)
├── getInitialStats()          → SystemStats
│
└── tickSimulation()           → {coins, agents, feed, launches, stats, triggeredEvent}
    │                           ↑ called every N ms by setInterval in App
    │
    └── App.tsx
        │
        ├── setCoins(nextState.coins)
        ├── setAgents(nextState.agents)
        ├── setFeed(nextState.feed)
        ├── setLaunches(nextState.launches)
        ├── setStats(nextState.stats)
        └── setLastTriggeredEvent(nextState.triggeredEvent)
            │
            └── React re-render → all memoized components check prop changes
```

**Memoization:** All 11 components are wrapped in `React.memo`. They re-render only when their props change (shallow comparison).

## 4. State Management

**No global store.** All state lives in `App.tsx` via `useState`:

```typescript
const [coins, setCoins] = useState<Coin[]>([])
const [agents, setAgents] = useState<Agent[]>([])
const [feed, setFeed] = useState<FeedEvent[]>([])
const [launches, setLaunches] = useState<LaunchItem[]>([])
const [stats, setStats] = useState<SystemStats>(...)
const [selectedCoinId, setSelectedCoinId] = useState<string>('')
const [selectedAgentId, setSelectedAgentId] = useState<string | null>(null)
const [speed, setSpeed] = useState<number>(1000)
const [isPaused, setIsPaused] = useState<boolean>(false)
```

**Downward data flow only.** Children receive data via props. No context, no reducers, no stores.

## 5. Rendering Architecture

### Canvas Components (2)
- **EcosystemMap:** Persistent `requestAnimationFrame` loop. Physics simulation runs continuously at 60fps. Node positions update in-place via refs, not via React state.
- **PriceChart:** Persistent `requestAnimationFrame` loop. Chart redraws on every frame. Data interpolation between old/new values over 350ms (cubic ease-out). Hover crosshair drawn in same frame.

### DOM Components (9)
- **LiveFeed, Leaderboard, etc.:** Standard React rendering. Wrapped in `React.memo` to prevent re-renders when unrelated state changes.

### CSS Animations
- `animate-ticker`: horizontal scroll via `translate3d`
- `animate-ping/pulse/spin`: indicator animations with `will-change` for GPU acceleration
- `.interactive`: unified hover/press system (scale + brightness)
- `.row-hover`: subtle lift on hover
- `@media (prefers-reduced-motion)`: disables all animations for accessibility

## 8. Known Limitations

1. **No real API** — all data is simulated client-side
2. **No persistence** — state resets on page refresh
3. **No error boundaries** — any JS error crashes the entire dashboard
4. **No mobile testing** — responsive classes exist but untested on real devices
5. **Memory leaks possible** — interval/rAF cleanup depends on proper useEffect returns
